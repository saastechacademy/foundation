# 🔐 Application Security Engineer (AppSec)

## 🎯 Core Mission
Ensure that applications are secure by design and implementation.

## 🧩 Key Responsibilities

### 1. Secure Code Review & Vulnerability Fixing
- Perform manual and automated code reviews
- Identify vulnerabilities such as:
    - XSS, SQL Injection, CSRF
    - Authentication & authorization flaws
- Guide developers on secure coding practices

---

### 2. Application Security Testing
- Use tools for:
    - SAST (Static Application Security Testing)
    - DAST (Dynamic Application Security Testing)
    - SCA (Software Composition Analysis for dependency vulnerabilities)
- Perform manual penetration testing (light to moderate)

---

### 3. Threat Modeling
- Analyze application architecture
- Identify:
    - Attack surfaces
    - Data flow risks
- Suggest mitigations early in development

---

### 4. Authentication & Authorization Security
- Review login systems and session management
- Ensure:
    - Strong password policies
    - Secure session handling
    - Proper RBAC/permission checks

---

### 5. API & Web Security
- Secure REST and GraphQL APIs
- Validate:
    - Input/output handling
    - Rate limiting
    - Token security (JWT, OAuth)

---

### 6. Security in SDLC (Shift-Left Security)
- Integrate security early in the development lifecycle
- Work closely with developers during design and coding phases

---

### 7. Vulnerability Management
- Triage vulnerabilities
- Prioritize fixes based on severity (CVSS)
- Track remediation lifecycle

---

### 8. Security Standards & Compliance
- Follow frameworks like:
    - OWASP Top 10
    - CWE (Common Weakness Enumeration)
- Ensure applications meet internal security policies

---

### 9. Developer Training & Awareness
- Conduct sessions on:
    - Secure coding
    - Common vulnerabilities
- Create internal security guidelines

---

## 🧠 Skill Focus
- Strong in programming and security
- Deep understanding of application internals
